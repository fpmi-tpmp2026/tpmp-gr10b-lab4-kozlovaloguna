## 1. Постановка задачи (Задание 1)

### 1.1. База данных «Абитуриент»

Необходимо создать базу данных для хранения информации об абитуриентах. Структура таблицы `Applicant`:

| Поле | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | INTEGER | Уникальный идентификатор (первичный ключ) |
| `last_name` | TEXT | Фамилия |
| `first_name` | TEXT | Имя |
| `middle_name` | TEXT | Отчество |
| `gender` | TEXT | Пол |
| `nationality` | TEXT | Национальность |
| `birth_date` | TEXT | Дата рождения (ГГГГ-ММ-ДД) |
| `postal_code` | TEXT | Почтовый индекс |
| `country` | TEXT | Страна |
| `region` | TEXT | Область |
| `district` | TEXT | Район |
| `city` | TEXT | Город |
| `street` | TEXT | Улица |
| `house` | TEXT | Дом |
| `apartment` | TEXT | Квартира |
| `ct_scores` | TEXT | Оценки по ЦТ (через запятую) |
| `passing_score` | REAL | Проходной балл |

### 1.2. Функциональные требования 

1.  Создание таблицы (`CREATE`).
2.  Вставка данных (`INSERT`).
3.  Выборка данных (`SELECT`) с сортировкой по `id`, по имени, вывод последних 5 строк (`LIMIT`).
4.  Фильтрация: `WHERE id = 5`.
5.  Поиск по маске: фамилия начинается на первую букву фамилии студента (`LIKE`).
6.  Переименование таблицы (`ALTER TABLE ... RENAME`).
7.  Обновление данных (`UPDATE`).
8.  Удаление строк по `id` и по названию (фамилии) объекта.
9.  Экспорт базы в файлы `.sql` и `.csv`.
10. Удаление таблицы (`DROP TABLE`).

### 1.3. Дополнительные запросы 

1.  Вывести названия и описания всех категорий расходов (`ExpenseCategory`).
2.  Вывести названия магазинов, даты покупок и суммы (`Spendings`), отсортированные по убыванию суммы.

## 2. Диаграммы UML (Моделирование системы)

Ниже представлены диаграммы, описывающие архитектуру и поведение системы, разработанные с использованием синтаксиса PlantUML.

### 2.1. Диаграмма вариантов использования (Use Case Diagram)

**Описание:** Диаграмма вариантов использования отображает функциональные требования к системе с точки зрения пользователя. Она показывает, какие действия могут выполнять акторы (Оператор, Абитуриент) при взаимодействии с системой.

#### Диаграмма

```plantuml
@startuml
left to right direction
actor "Абитуриент" as Applicant
actor "Оператор ПК" as Operator
actor "Система" as System

rectangle "Управление базой данных «Абитуриент»" {
  usecase "Добавить абитуриента" as UC1
  usecase "Редактировать данные" as UC2
  usecase "Удалить абитуриента" as UC3
  usecase "Просмотреть список" as UC4
  usecase "Фильтровать по баллу" as UC5
  usecase "Поиск по маске (LIKE)" as UC6
  usecase "Экспорт (.sql, .csv)" as UC7
  usecase "Просмотр категорий расходов" as UC8
  usecase "Просмотр покупок (сортировка)" as UC9
}
