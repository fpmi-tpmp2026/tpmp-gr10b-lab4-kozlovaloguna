# Отчет по проектированию системы управления БД (SQLite + C)

## 1. Введение
Данный проект представляет собой консольное приложение на языке C, предназначенное для администрирования базы данных сотрудников (`people.db`). Основная цель системы — обеспечить интерфейс для выполнения CRUD-операций, работы с бинарными объектами (фотографиями) и управления транзакциями через библиотеку SQLite.

---

## 2. Описание диаграммы вариантов использования (Use Case Diagram)
Диаграмма описывает взаимодействие внешнего пользователя с функциональными возможностями системы. Основным актером является **Пользователь**, который инициирует следующие сценарии:
*   **Добавить человека**: Внесение данных о сотруднике (имя, фамилия, ID позиции) в таблицу.
*   **Найти по ID**: Получение полных данных о человеке по его уникальному идентификатору.
*   **Найти по фамилии (LIKE)**: Фильтрация записей по частичному совпадению текста.
*   **Выгрузить фото в файл**: Извлечение данных формата BLOB из базы и сохранение их в виде изображения.
*   **Тест транзакций**: Демонстрация разницы в скорости вставки данных с использованием механизмов BEGIN/COMMIT.

---

## 3. Описание диаграммы модулей (Class Diagram / Module Structure)
Проект построен по модульному принципу, что обеспечивает разделение ответственности:
*   **Модуль управления (main.c)**: Точка входа в программу. Реализует бесконечный цикл обработки команд и диалог с пользователем.
*   **Модуль данных (database.c)**: Содержит всю логику работы с SQLite API (подготовка стейтментов, привязка параметров, обработка шагов выполнения SQL).
*   **Заголовочный файл (database.h)**: Выступает в роли интерфейса, предоставляя прототипы функций для вызова из главного модуля.

---

## 4. Описание диаграммы деятельности (Activity Diagram)
Диаграмма деятельности визуализирует логику работы приложения:
1.  **Старт**: Инициализация кодировки UTF-8 для корректного отображения кириллицы и попытка открытия файла БД.
2.  **Цикл**: Программа переходит в состояние ожидания выбора пользователя.
3.  **Ветвление**: Оператор `switch` направляет поток управления к нужной функции (поиск, вставка или экспорт).
4.  **Завершение**: При выборе команды выхода программа закрывает соединение с SQLite и освобождает ресурсы.

---

## 5. Описание диаграммы последовательности (Sequence Diagram)
Диаграмма показывает динамику взаимодействия компонентов при поиске данных:
*   Пользователь инициирует поиск через `main.c`.
*   `main.c` вызывает функцию из `database.c`.
*   Модуль данных взаимодействует с движком **SQLite Engine**, выполняя последовательность команд: `prepare` (компиляция SQL), `bind` (передача параметров), `step` (выполнение) и `finalize` (очистка).
*   Результат возвращается пользователю через стандартный поток вывода.

---

## 6. Описание диаграммы компонентов (Component Diagram)
Диаграмма отображает физическую организацию проекта:
*   **Компоненты кода**: Исходные файлы `main.c`, `database.c` и интерфейс `database.h`.
*   **Внешние библиотеки**: Зависимость от `sqlite3` (заголовочный файл и скомпилированная библиотека).
*   **Артефакты сборки**: Итоговый исполняемый файл `app.exe`, который взаимодействует с внешним файлом базы данных `people.db`.

---

## 7. Заключение
Выбранная архитектура обеспечивает четкое разделение между интерфейсом и логикой доступа к данным. Использование подготовленных запросов (Prepared Statements) гарантирует безопасность и высокую производительность приложения.
