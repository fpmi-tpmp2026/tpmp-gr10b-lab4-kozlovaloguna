
# Цветочная Оранжерея — Система управления заказами

## Описание проекта

Информационная система для автоматизации работы цветочной оранжереи. Проект позволяет управлять каталогом цветов, формировать композиции, обрабатывать заказы покупателей и формировать отчётность.

## Функциональные возможности

- **Управление каталогом цветов** — добавление, редактирование, удаление информации о цветах
- **Создание композиций** — формирование букетов из различных видов цветов
- **Обработка заказов** — приём заказов с учётом срочности выполнения
- **Расчёт стоимости** — автоматический расчёт с учётом наценки за срочность
- **Формирование отчётов** — аналитика по продажам и использованию цветов

## Система тарифов

| Срок выполнения | Наценка |
|-----------------|---------|
| 1 сутки         | +25%    |
| 2 суток         | +15%    |
| Обычный заказ   | 0%      |

## Структура базы данных

### Таблицы

| Таблица | Описание |
|---------|----------|
| `flowers` | Справочник цветов (название, сорт, цена) |
| `compositions` | Каталог композиций |
| `composition_items` | Состав композиций (связь цветов и композиций) |
| `orders` | Заказы покупателей |
| `order_costs` | Расчёт стоимости заказов |

## UML-диаграммы

Проект включает следующие диаграммы:

### 1. Диаграмма вариантов использования
Описывает функциональные требования системы с точки зрения актёров (Покупатель, Администратор).

### 2. Диаграмма классов
Описывает структуру приложения, основные сущности и их взаимосвязи.

### 3. Диаграмма деятельности
Иллюстрирует бизнес-процесс обработки заказа от принятия до выдачи.

### 4. Диаграмма последовательности
Показывает взаимодействие объектов при формировании и выполнении заказа.

### 5. Диаграмма компонентов
Отображает архитектуру системы на уровне компонентов.

### 6. Диаграмма размещения
Описывает физическое развёртывание системы на серверах.
